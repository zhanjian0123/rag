# Web RAG 知识库系统

一个智能知识库系统，支持多源知识输入（网页 URL、PDF、Word 文档），提供智能切片、混合检索、查询重写与转换等高级 RAG 功能。

## ✨ 特性

### 核心特性
- **多源统一** - 支持网页、PDF、Word 文档等多种数据源
- **一键入库** - 自动提取内容、切片、向量化
- **智能问答** - 基于混合检索和 LLM 的智能问答
- **无需 API** - 不依赖任何平台的 API 接口
- **增量更新** - 支持已有内容的定期刷新

### 高级特性
- **多策略切片** - 支持句子、语义、Markdown 结构等多种切片方式
- **混合检索** - 结合向量检索 + BM25 关键词检索
- **查询优化** - 查询重写与转换，提升检索准确率
- **多路召回** - 多策略检索结果融合

## 🛠️ 技术栈

### 后端
| 类别 | 技术 |
|------|------|
| **框架** | FastAPI |
| **数据库** | SQLite (SQLAlchemy) |
| **向量数据库** | ChromaDB |
| **网页提取** | Trafilatura |
| **PDF 解析** | PyMuPDF / pdfplumber |
| **Word 解析** | python-docx |
| **分词/关键词** | jieba / rank-bm25 |
| **LLM/Embedding** | OpenAI API / DashScope API |

### 前端
| 类别 | 技术 |
|------|------|
| **框架** | Next.js 14 (React + TypeScript) |
| **样式** | Tailwind CSS |
| **构建工具** | PostCSS |

### 1. 文档管理
| 功能 | 说明 |
|------|------|
| **网页 URL** | 输入 URL 自动抓取并提取内容 |
| **PDF 上传** | 上传本地 PDF 文件，解析文本和元数据 |
| **Word 上传** | 上传 .docx 文件，解析文档内容 |
| **批量导入** | 支持批量上传多个文件 |

### 2. 知识库管理
| 功能 | 说明 |
|------|------|
| **集合管理** | 创建/编辑/删除知识库集合 |
| **文档管理** | 查看集合内文档，管理处理状态 |
| **构建任务** | 异步处理任务，进度实时跟踪 |
| **增量更新** | 检测内容变化，增量向量化 |

### 3. 智能切片 🆕
支持多种切片策略，可按文档类型自动选择或手动配置：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **句子切片** | 按句子边界切分，保持语义完整 | 新闻、文章 |
| **段落切片** | 按自然段落切分 | 博客、报告 |
| **语义切片** | 使用 LLM 识别语义边界 | 复杂文档 |
| **Markdown 切片** | 按标题层级切分 | 技术文档、MD 文件 |
| **固定长度** | 固定字符数 + 重叠 | 通用场景 |
| **混合切片** | 多种策略组合 | 混合文档类型 |

### 4. 混合检索 🆕
结合多种检索方式，提升召回率：

| 检索方式 | 说明 |
|----------|------|
| **向量检索** | 语义相似度匹配 |
| **BM25 检索** | 关键词匹配 |
| **元数据过滤** | 按时间、来源等筛选 |
| **混合融合** | 加权融合多种检索结果 |

### 5. 查询优化 🆕
| 功能 | 说明 |
|------|------|
| **查询重写** | LLM 重写用户问题，明确意图 |
| **查询扩展** | 添加同义词、相关词 |
| **子查询分解** | 复杂问题拆分为多个子查询 |
| **假设文档嵌入** | HyDE 策略，生成假设答案辅助检索 |

### 6. 智能问答
| 功能 | 说明 |
|------|------|
| **基础问答** | 基于检索内容生成回答 |
| **流式输出** | 打字机效果，实时显示 |
| **来源引用** | 标注参考来源和页码 |
| **多轮对话** | 支持上下文多轮问答 |

## 📄 切片策略详解

### 句子切片
```python
# 按句子边界切分，适合保持语义完整
文本 → [句子 1] [句子 2] [句子 3] → 合并为 chunks
```

### 语义切片
```python
# 使用 LLM 识别语义主题变化
文本 → LLM 分析 → [主题 1] [主题 2] [主题 3] → chunks
```

### Markdown 切片
```python
# 按标题层级切分
# Heading 1
## Heading 2
### Heading 3
→ 按层级分组 → chunks
```
